Create skills/seo-site-audit-pro/SKILL.md — the most complex skill in the project. Uses INLINE tool calls (NOT subagents — see research recommendation #4) in sequential waves with checkpoint saves to disk.

YAML frontmatter:
- name: seo-site-audit-pro
- description: Trigger phrases include "site audit pro", "full audit", "comprehensive audit", "audit with live data", "deep audit", "pro audit", "complete SEO audit". Keep under 500 chars.
- allowed-tools: [Read, Bash, ToolSearch, Write]

Structure:

1. ## References — @skills/seo/references/mcp-degradation.md, @skills/seo/references/ahrefs-api-reference.md, @skills/seo/references/gsc-api-reference.md

2. ## Inputs:
   - `domain`: Bare domain to audit (required), e.g. `example.com`
   - `site` (optional): GSC property URL to include GSC data in the audit

3. ## Important: API Usage Warning
   Add a MANDATORY user-facing warning at the start of execution:
   "**API Usage Notice:** This command calls approximately 10-12 Ahrefs API tools sequentially. Each call consumes API units from your Ahrefs account. You can stop after any wave if budget is a concern. Estimated waves: 3 (each takes 1-2 minutes)."
   This addresses STATE.md blocker: "Ahrefs API unit cost per endpoint unknown."

4. ## MCP Check:
   - ToolSearch "+ahrefs" — required. If unavailable, stop.
   - ToolSearch "+google-search-console" — optional. If unavailable AND site param provided, note "GSC unavailable — audit will use Ahrefs data only" and continue.

5. ## Architecture:
   "This command uses SEQUENTIAL wave execution with checkpoint saves. Each wave runs its tool calls one at a time. Results are saved to a checkpoint file after each wave before the next begins. This prevents cascade failures from rate-limit errors (GitHub Issue #6594)."

   DO NOT run tool calls in parallel. DO NOT spawn subagents. All tool calls are inline within this single skill execution.

6. ## Execution:

   **Step 0 — Initialize Checkpoint File:**
   Use Bash to create checkpoint filename: `seo-audit-pro-checkpoint-{domain}.md`
   Resolve absolute path in current working directory.
   Use Write tool to create initial checkpoint: "# Site Audit Pro: {domain}\n**Started:** {date}\n**Status:** In progress\n"

   **Wave 1 — Domain Authority and Backlink Health (4 tool calls):**

   Call each tool ONE AT A TIME. For each call:
   - If success → store result
   - If rate-limit or error → log "SKIPPED: {tool-name} — {error reason}" and continue to next tool
   - NEVER stop the wave because one tool failed

   Tool calls in order:
   a) `mcp__claude_ai_ahrefs__site-explorer-metrics` with target={domain}
   b) `mcp__claude_ai_ahrefs__site-explorer-domain-rating` with target={domain}
   c) `mcp__claude_ai_ahrefs__site-explorer-all-backlinks` with target={domain} (top 20 results)
   d) `mcp__claude_ai_ahrefs__site-explorer-broken-backlinks` with target={domain}

   **Checkpoint Save:** Use Write tool to APPEND Wave 1 results to checkpoint file.
   Output to user: "Wave 1 complete (Domain Authority & Backlinks). Checkpoint saved. Starting Wave 2..."

   **Wave 2 — Keyword and Competitive Position (3-4 tool calls):**

   a) `mcp__claude_ai_ahrefs__site-explorer-organic-keywords` with target={domain} (top 20)
   b) `mcp__claude_ai_ahrefs__site-explorer-organic-competitors` with target={domain}
   c) `mcp__claude_ai_ahrefs__keywords-explorer-matching-terms` — use top competitor from (b) to find content gaps. Use ToolSearch to verify parameter schema first.
   d) IF `site` param provided AND GSC available: call GSC `query_search_analytics` for domain overview (top 20 queries by clicks)

   **Checkpoint Save:** Use Write tool to APPEND Wave 2 results to checkpoint file.
   Output to user: "Wave 2 complete (Keywords & Competition). Checkpoint saved. Starting Wave 3..."

   **Wave 3 — Content and Technical Insights (3-4 tool calls):**

   a) IF GSC available: GSC opportunities query (high impressions, low CTR)
   b) IF GSC available: GSC drops query (pages losing traffic in last 28 days)
   c) `mcp__claude_ai_ahrefs__site-explorer-top-pages` with target={domain} (top 10)
   d) `mcp__claude_ai_ahrefs__site-explorer-anchors` with target={domain}

   **Checkpoint Save:** Use Write tool to APPEND Wave 3 results to checkpoint file.
   Output to user: "Wave 3 complete (Content & Technical). All waves done. Generating final report..."

   **Step Final — Cross-MCP Synthesis and Final Report:**

   Read all checkpoint data. Generate a comprehensive audit report with these sections:

   a) Executive Summary — AI-generated 3-5 sentence analysis of the domain's overall SEO health
   b) Domain Authority Profile — DR, backlinks, referring domains from Wave 1
   c) Backlink Health — Top backlinks + broken links for reclamation from Wave 1
   d) Keyword Portfolio — Top keywords, competition landscape from Wave 2
   e) Content Gaps — Keywords competitors rank for but this domain doesn't, from Wave 2
   f) GSC Performance (only if GSC data available) — Overview, opportunities, drops from Waves 2-3
   g) Top Pages Analysis — Best performing pages from Wave 3
   h) Anchor Text Profile — Distribution analysis from Wave 3
   i) **Cross-MCP Insights (ONLY when both Ahrefs AND GSC data present):**
      Synthesize findings that require both data sources. Examples:
      - "You rank #{position} for '{keyword}' per GSC but Ahrefs shows 3 competitors with DR 20+ points higher — priority opportunity"
      - "GSC shows traffic declining for {page} but Ahrefs shows 5 new backlinks this month — possible algorithm volatility, not link loss"
      - "Top organic keyword per GSC has {clicks} clicks but Ahrefs estimates {traffic} — gap suggests click-through optimization opportunity"
   j) Skipped Data Sources — List any tools that were skipped due to rate limits or errors
   k) Recommendations — Prioritized action items based on all findings

   CRITICAL: Monetary conversion — divide all Ahrefs monetary values by 100 for USD display.

7. ## Output Format — Full report printed to terminal (unlike seo-report which saves to file). The checkpoint file remains on disk for reference.

8. ## Error sections for Ahrefs unavailable (stop) and GSC unavailable (continue Ahrefs-only)
