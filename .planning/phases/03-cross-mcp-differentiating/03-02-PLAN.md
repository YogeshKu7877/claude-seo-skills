Structure:
1. ## References — @skills/seo/references/mcp-degradation.md, @skills/seo/references/ahrefs-api-reference.md
2. ## Inputs:
   - `brand`: The brand name or domain to monitor (required). Can be a company name ("Notion") or domain ("notion.so").

   - ToolSearch "+ahrefs" — required, NO fallback. If unavailable, display error and stop.

   - Step 1 — Schema Discovery (CRITICAL): Use ToolSearch with query "brand-radar" to discover available brand-radar tools and their parameter schemas. Inspect the returned tool definitions to determine:

   - If any individual tool call fails (rate limit, parameter error), log "SKIPPED: {tool-name} — {error}" and continue with remaining tools. Only stop entirely if the primary mentions-overview call fails.

```
### Share of Voice
{Data from brand-radar-sov-overview}

### Top Cited Sources in AI Responses
{Table from brand-radar-cited-domains — domains that get cited alongside this brand}

{AI interpretation of brand visibility trends and competitive position}
```

Note in Output Format: "Output tables are adapted at runtime based on discovered API response fields. The exact columns depend on what the Brand Radar API returns."

<task type="auto">
  <name>Task 2: Create seo-report/SKILL.md with 4 report types and file persistence</name>
  <files>skills/seo-report/SKILL.md</files>
  <action>
Create skills/seo-report/SKILL.md following the Phase 2 sub-skill pattern, extended with file persistence via Write tool.

YAML frontmatter:
- name: seo-report
- description: Trigger phrases include "SEO report", "generate report", "monthly report", "weekly report", "audit report", "competitor report", "save report", "export SEO data". Keep under 500 chars.
- allowed-tools: [Read, Bash, ToolSearch, Write]

Note: `Write` is added to allowed-tools for the first time in any sub-skill — this is the file persistence requirement from CROSS-05.

Structure:
1. ## References — @skills/seo/references/mcp-degradation.md, @skills/seo/references/ahrefs-api-reference.md, @skills/seo/references/gsc-api-reference.md
2. ## Inputs:
   - `type`: Report type — one of: `monthly`, `weekly`, `audit`, `competitor` (required)
   - `domain`: The domain to report on, bare domain e.g. `example.com` (required)
   - `site` (optional): GSC property URL for including GSC data
   - `competitor` (optional, required for competitor type): Competitor domain to compare against
   - `output_dir` (optional): Directory to save report file. Default: current working directory.
3. ## MCP Check:
   - ToolSearch "+ahrefs" — required for all report types. If unavailable, stop.
   - ToolSearch "+google-search-console" — optional, checked only if `site` param provided. Note missing GSC in report if unavailable.
4. ## Execution:
   - Step 0 — Generate filename and resolve output path:
     Use Bash: `echo "$(pwd)/seo-report-$(date +%Y-%m-%d)-{domain}-{type}.md"`
     If output_dir provided, use that instead of pwd.
     Store absolute path for later.

   - Step 1 — Gather Data (varies by report type):

     **Monthly report:**
     a) Ahrefs overview: site-explorer-metrics + domain-rating
     b) Ahrefs keywords: site-explorer-organic-keywords (top 20)
     c) Ahrefs DR history: site-explorer-domain-rating-history (last 12 months)
     d) GSC compare (if site provided): query_search_analytics for current vs previous month

     **Weekly report:**
     a) GSC drops (if site provided): query_search_analytics comparing last 7 days vs prior 7 days
     b) GSC opportunities (if site provided): high-impression low-CTR queries
     c) Ahrefs new-links: site-explorer-referring-domains (last 7 days)

     **Audit report:**
     a) Ahrefs overview: site-explorer-metrics + domain-rating
     b) Ahrefs backlinks: site-explorer-all-backlinks (top 20)
     c) Ahrefs broken-links: site-explorer-broken-backlinks
     d) Ahrefs anchor-analysis: site-explorer-anchors
     e) Ahrefs top-pages: site-explorer-top-pages (top 10)

     **Competitor report:**
     a) Ahrefs overview for both domains
     b) Ahrefs competitors: site-explorer-organic-competitors
     c) Ahrefs content-gap: keywords-explorer-matching-terms filtered by competitor
     d) Ahrefs anchor-analysis for both domains

     For each tool call: if rate-limit error, log "Data unavailable: {tool} — rate limit" and continue. Report notes all skipped data sources.

   - Step 2 — Generate AI Executive Summary:
     Before assembling the report, analyze ALL gathered data and write an executive summary paragraph (3-5 sentences) that interprets the data patterns, identifies the most important finding, and suggests one priority action. This is NOT a raw data dump — it is Claude's analytical interpretation. Example: "Domain authority has grown from DR 45 to DR 52 over the past 6 months, driven primarily by 340 new referring domains. However, organic traffic declined 12% month-over-month, suggesting a potential Google algorithm impact. Priority: Investigate the traffic drop by reviewing /seo gsc drops for affected pages."

   - Step 3 — Assemble Report:
     Build full markdown document with sections appropriate to report type. Structure:
     ```
     # SEO {Type} Report: {domain}
     **Generated:** {date}
     **Report Type:** {type}
     **Data Sources:** {list MCPs used}

     ## Executive Summary
     {AI-generated narrative from Step 2}

     ## {Section per data source gathered}
     {Tables and analysis}

     ---
     *Generated by /seo report — Claude SEO Skill*
     *Data sources: {Ahrefs, GSC, or both}*
     ```

   - Step 4 — Save Report to Disk:
     Use Write tool with `file_path` set to the absolute path from Step 0 and `content` set to the full report markdown.

   - Step 5 — Confirm to User:
     Output: "Report saved to: {absolute_file_path}"
     Include summary: "{N} sections covering {data sources used}"
     CRITICAL: Always print the FULL ABSOLUTE PATH so the user can find the file.

5. ## Output Format — The terminal output is just the save confirmation. The report itself is in the saved file.
6. ## Error sections for Ahrefs unavailable (stop) and GSC unavailable (continue with Ahrefs-only, note in report)

IMPORTANT DESIGN NOTES:
- This is the FIRST skill in the project that uses the Write tool for file persistence
- The report is saved to disk, NOT printed to terminal (CROSS-05 success criterion)
- Each report type has different data requirements — the skill must route to the correct tool set based on `type` parameter
- CRITICAL monetary conversion applies to ALL Ahrefs monetary fields in the report
