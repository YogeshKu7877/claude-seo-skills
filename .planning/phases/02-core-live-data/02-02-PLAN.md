<!-- Each sub-skill uses ToolSearch "+gsc" or "+search_analytics" to find tools -->

GSC Tool: query_search_analytics
  Required: siteUrl, startDate, endDate
  Optional: dimensions (["query"], ["page"], ["query","page"]), rowLimit (default 1000), filters, account
  Returns: rows with clicks, impressions, ctr (decimal), position per dimension value

GSC Tool: compare_performance
  Required: siteUrl, currentStartDate, currentEndDate, previousStartDate, previousEndDate
  Optional: dimension ("page" or "query"), limit, account
  Returns: rows with current/previous metrics, clicksChange, clicksChangePercent, positionChange

GSC Tool: find_keyword_opportunities
  Required: siteUrl, startDate, endDate
  Optional: minImpressions (default 100), maxCtr (default 0.05), maxPosition (default 20), account
  Returns: keywords with high impressions but low CTR — ready-made opportunity list

GSC Tool: inspect_url
  Required: siteUrl, inspectionUrl
  Optional: account
  Returns: coverageState, indexingState, pageFetchState, robotsTxtState, lastCrawlTime

GSC Tool: analyze_brand_queries
  Required: siteUrl, startDate, endDate, brandTerms (array of strings)
  Optional: account
  Returns: branded vs non-branded traffic split with metrics for each

GSC Tool: list_sites
  Optional: account
  Returns: list of GSC properties with permission level

GSC Tool: get_top_pages
  Required: siteUrl, startDate, endDate
  Optional: sortBy, limit, account
  Returns: top pages by the specified metric

Conventions:
- CTR returned as decimal (0.0523) — multiply by 100 for display (5.23%)
- Dates as YYYY-MM-DD strings
- GSC data has ~3 day delay — use today-3 as endDate
- Property format: both sc-domain:example.com and https://example.com accepted
- Tool name prefix: UNKNOWN — discover via ToolSearch at runtime

MCP Check Pattern (from mcp-degradation.md):
1. Use ToolSearch with query "+gsc" (or "+google-search-console" or "+search_analytics")
2. If tools returned → note prefix, proceed
3. If no tools → display GSC error template from mcp-degradation.md, stop

**1. seo-gsc-overview/SKILL.md** (GSC-01)
- Description triggers: "GSC overview", "search console performance", "how is my site doing in Google", "clicks and impressions"
- Calls `query_search_analytics` twice: once with dimensions=["query"] rowLimit=25, once with dimensions=["page"] rowLimit=25
- Also calls `get_top_pages` for a top-pages breakdown
- Output: Summary table (Total Clicks, Total Impressions, Avg CTR, Avg Position), Top 25 Queries table, Top 25 Pages table
- Date range: last 28 days (endDate = today-3, startDate = endDate-28)
- CTR display: multiply decimal by 100, format as X.XX%

**2. seo-gsc-drops/SKILL.md** (GSC-02)
- Description triggers: "GSC drops", "ranking drops", "lost traffic", "pages losing rankings", "keyword drops"
- Calls `compare_performance` with dimension="page" and dimension="query" (two calls)
- Current period: last 28 days; Previous period: 28 days before that
- Filter results to show only rows with negative clicksChange
- Sort by clicksChange ascending (biggest losses first)
- Output: Top dropping pages table (URL, Current Clicks, Previous Clicks, Change, Change%), Top dropping queries table (same columns)
- Highlight rows with >50% drop as CRITICAL

**3. seo-gsc-opportunities/SKILL.md** (GSC-03)
- Description triggers: "GSC opportunities", "quick wins", "low CTR keywords", "high impressions low clicks", "easy SEO wins"
- Calls `find_keyword_opportunities` — this is a dedicated tool that does the filtering
- Default params: minImpressions=100, maxCtr=0.05 (5%), maxPosition=20
- Output: Opportunity table (Keyword, Impressions, Clicks, CTR, Position, Potential)
- Add "Potential" column: if position 4-10 and CTR<3%, note "Title/meta optimization"; if position 11-20, note "Content expansion"
- Sort by impressions descending (highest opportunity first)

**4. seo-gsc-cannibalization/SKILL.md** (GSC-04)
- Description triggers: "keyword cannibalization", "same keyword multiple pages", "cannibalizing keywords", "duplicate rankings"
- Calls `query_search_analytics` with dimensions=["query","page"], rowLimit=1000
- Post-process: group by query, find queries that appear with 2+ different pages
- For each cannibalized query, show all competing pages with their respective clicks, impressions, CTR, position
- Output: Cannibalization report table grouped by keyword — highlight the "winning" page (most clicks) and "losing" pages
- Add recommendation: consolidate content or add canonical signals

**5. seo-gsc-indexing/SKILL.md** (GSC-05 — maps to REQUIREMENTS.md `index-issues`)
- Description triggers: "index issues", "indexing problems", "not indexed", "indexing coverage", "why is my page not indexed"
- CRITICAL: `inspect_url` is rate-limited. Do NOT call it for every page on the site.
- Strategy: First call `query_search_analytics` with dimensions=["page"], rowLimit=100, sorted by impressions. This gives pages GSC knows about. Then call `inspect_url` on up to 20 pages — prioritize pages with lowest impressions (most likely to have indexing issues) or pages the user specifically asks about.
- If user provides specific URLs to check, inspect those directly (still cap at 20).
- Output: Indexing Status table (URL, Coverage State, Indexing State, Robots.txt State, Last Crawl Date)
- Group results by coverage state (Indexed, Excluded, Error)

- Call twice: dimension="page" and dimension="query"
- Output: Summary deltas table (Clicks Change, Impressions Change, CTR Change, Position Change), Top movers by page, Top movers by query

**7. seo-gsc-brand-vs-nonbrand/SKILL.md** (GSC-07)
- Description triggers: "brand vs nonbrand", "branded traffic", "brand queries", "non-brand traffic", "branded vs generic"
- CRITICAL: This tool REQUIRES user-provided brand terms. The `analyze_brand_queries` tool needs a `brandTerms` array.
- Execution flow:
  1. Ask the user for their brand terms BEFORE making any API call. Pre-populate suggestion: extract likely brand from the domain name (e.g., `example.com` → suggest ["example"]). Present as: "I'll analyze brand vs non-brand traffic. Please confirm your brand terms: `[suggested terms]`. Add or remove terms as needed."
  2. Wait for user confirmation or modification.
  3. Call `analyze_brand_queries` with confirmed brandTerms.
- Output: Brand Split table (Branded Clicks, Branded Impressions, Non-Brand Clicks, Non-Brand Impressions, Brand % of Total), Top branded queries, Top non-brand queries
- Add insight: if brand traffic >70%, note dependency risk; if <10%, note brand awareness opportunity

**8. seo-gsc-content-decay/SKILL.md** (GSC-08)
- Description triggers: "content decay", "declining pages", "losing rankings", "pages losing traffic over time", "90 day decline"
- Calls `compare_performance` with 90-day window: current=last 90 days, previous=90 days before that
- Filter for rows with negative clicksChange AND negative impressionsChange (true decay, not seasonal)
- Sort by clicksChange ascending (biggest decays first)
- Output: Content Decay table (URL, Current Clicks, Previous Clicks, Change%, Current Position, Previous Position, Position Change)
- Add "Decay Severity" column: >50% drop = CRITICAL, 30-50% = HIGH, 10-30% = MEDIUM
- Add recommendation per page: "Update content", "Check for technical issues", "Review competitor content"

**9. seo-gsc-new-keywords/SKILL.md** (GSC-09)

- Also include rows where previous impressions were 0 but current impressions >0 (just started appearing)
- Sort by current clicks descending (most impactful new keywords first)

- Add insight: if many new keywords cluster around a topic, note the emerging content opportunity
